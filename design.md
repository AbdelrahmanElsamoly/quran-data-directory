# Design Guidelines

Use this file as the design baseline for pages worked on in this project.

## Page Layout

- Start each page with the actual user experience or primary content, not a marketing-style landing section unless the page is specifically meant to be promotional.
- Keep layouts clear, calm, and easy to scan. Quran data, resource pages, dashboards, and directory views should feel focused and trustworthy.
- Use full-width page sections with constrained inner content. Avoid placing large page sections inside floating cards.
- Keep navigation, filters, lists, and detail panels predictable across pages.

## Spacing And Gaps

- Use consistent vertical gaps between major page sections.
- Use smaller, tighter gaps inside cards, list rows, filters, and compact tool areas.
- Avoid uneven or accidental-looking spacing between headings, body text, buttons, and repeated items.
- When a page has a grid, keep the row and column gaps consistent across desktop and mobile layouts.
- Make sure responsive breakpoints preserve readable spacing without creating large empty areas or crowded content.

## Components

- Use existing project components and styling patterns before introducing new ones.
- Keep cards for repeated items, modals, and genuinely framed content.
- Avoid nested cards unless the existing design system already uses that pattern.
- Buttons should have clear labels or familiar icons, and controls should match their purpose.

## Typography

- Match text size to the container. Use large display text only for true page heroes.
- Keep headings, labels, and body text readable on mobile and desktop.
- Do not let long words, labels, or translated text overflow their containers.

## Visual Style

- Prefer restrained, useful visual polish over decoration.
- Avoid one-color themes that make the page feel flat or overly branded by a single hue.
- Use images or visual assets when they help users understand the page, product, place, resource, or content being shown.

## Responsive Quality

- Check pages at mobile and desktop widths.
- Confirm text, buttons, cards, grids, and media do not overlap.
- Keep fixed-format elements such as toolbars, counters, tiles, and grids dimensionally stable so hover states or dynamic content do not shift the layout.
